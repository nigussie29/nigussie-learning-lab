# Course Blueprint

**Version:** 1.0

**Status:** Institutional Standard

---

# Purpose

The Course Blueprint defines the standard structure for every course offered by SkillBridge Academy.

Every course, regardless of subject, must follow this blueprint to ensure consistency, quality, and alignment with the SkillBridge Method.

---

# Course Design Philosophy

Every course should answer one fundamental question:

> **What meaningful problem will learners be able to solve after completing this course?**

Courses exist to develop creators, researchers, innovators, and leaders—not merely to transfer information.

---

# Course Identity

Every course must define:

- Course Title
- Category
- Description
- Level (Beginner, Intermediate, Advanced)
- Duration
- Prerequisites
- Target Audience

---

# Problem Statement

Every course begins by clearly defining:

- What problem exists?
- Why is this problem important?
- Who experiences this problem?
- Why should learners care?

---

# Learning Outcomes

By the end of the course, learners should be able to:

- Explain key concepts.
- Solve authentic problems.
- Build a real project.
- Apply knowledge in practical settings.
- Conduct research.
- Communicate their findings.

Learning outcomes must be measurable.

---

# The SkillBridge Learning Cycle

Every course follows the same educational sequence.

1. Learn
2. Practice
3. Build
4. Master
5. Apply
6. Research
7. Create
8. Lead

This sequence is mandatory.

---

# Course Structure

Each course contains:

- Modules
- Lessons
- Practice Activities
- Build Projects
- Mastery Assessments
- Application Challenges
- Research Extensions
- Portfolio Artifacts
- Final Capstone Project

---

# Mastery Pyramid Alignment

Every module should contribute to one or more levels of the Mastery Pyramid.

The goal is not simply to complete content but to progress toward mastery.

---

# Applied Mathematics Connection

Every course should explain the mathematical ideas that support the subject whenever appropriate.

Learners should understand not only how a solution works but why it works.

---

# Artificial Intelligence Integration

Artificial Intelligence should be used to:

- personalize learning,
- provide feedback,
- recommend resources,
- encourage critical thinking.

AI should support human reasoning rather than replace it.

---

# Research Integration

Every course must include research opportunities.

Examples include:

- Research questions
- Literature reviews
- Experiments
- Data analysis
- Reflection
- Innovation proposals

Research develops curiosity and lifelong learning.

---

# Portfolio Evidence

Every course must produce evidence of learning.

Examples include:

- Projects
- Code
- Dashboards
- Reports
- Presentations
- Research papers
- Videos
- Technical documentation

Portfolios demonstrate mastery.

---

# Assessment Philosophy

Assessment measures growth rather than memorization.

Evaluation should consider:

- Understanding
- Problem solving
- Creativity
- Communication
- Research
- Collaboration
- Reflection

---

# Career Connection

Every course should explain:

- Careers related to the subject
- Required professional skills
- Industry tools
- Portfolio expectations
- Future learning pathways

Learners should understand how today's work prepares them for tomorrow's opportunities.

---

# Role of Luminery

Luminery supports learners by:

- identifying strengths,
- recommending next steps,
- encouraging reflection,
- promoting mastery,
- connecting learning across courses.

Luminery is a mentor, not merely an assistant.

---

# Course Completion

Completion is not the goal.

Mastery is the goal.

A learner successfully completes a course by demonstrating the ability to understand, build, apply, research, create, and communicate knowledge.

---

# Success Question

Every course designer should ask:

> **Will this course help learners become creators, researchers, innovators, and leaders?**

If the answer is yes, the course aligns with the SkillBridge Method.

If the answer is no, redesign the course.

---

## SkillBridge Academy Principle

Every course exists to transform learners—not simply to deliver content.

Courses should inspire curiosity, develop mastery, encourage research, and prepare learners to improve the world.
